# Contributing to ProHire

First off, thank you for considering contributing to ProHire! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

### Reporting Bugs
If you find a bug (like a PHP error or a CSS glitch), please open an Issue:
1. Check if the issue has already been reported.
2. If not, create a new issue with a clear title and a detailed description of the problem.
3. Include steps to reproduce the bug and screenshots if possible.

### Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it!
1. Open an issue and label it as an "Enhancement."
2. Explain the feature and why it would be useful for the job portal.

### Pull Requests
Ready to contribute code? Follow these steps:
1. **Fork** the repository to your own account.
2. Create a new **Branch** for your feature (`git checkout -b feature/AmazingFeature`).
3. **Commit** your changes with clear messages (`git commit -m 'Add some AmazingFeature'`).
4. **Push** to the branch (`git push origin feature/AmazingFeature`).
5. Open a **Pull Request** against the `main` branch.
